Cross-Site Request Forgery.
Злоумылнениик подделывает форму с адресом нашего сервера, тем самым бразуер отправляет куки и вредные данные с форму(перевести деньги куда-то)
CSRF - добавляет одноразовый токен, который оптравляет сервер клиенту
csrf токен передается в параметрах ответа! через ключ _csrf

HTTP - stateless протокол - не сохраняет состояние
Сессия - это объект на сервера, хранящий информацию о клиенте

в Сессию можно добавить атрибуты и читать их на клиенте

Cookies - информация на бразуере клиента, которую он посылает каждый раз при обращении к серверу

у куки есть срок действия
у куки есть поля domain И path - к какому сайту/домену они относятся(same origin policy)
set-coockie - заголовок, в котором сервер передает куки для клиента
jsessiondID - куки уникальный идентификатор пользователя
у куки надо устанваливать срок действия, удалить куки - установить время жизни 0 или -1(при -1 куки удаляться при закрытии браузера)
